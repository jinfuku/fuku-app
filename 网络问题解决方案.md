# Git 推送网络问题解决方案

## 🔍 问题分析

您遇到的错误：
```
fatal: unable to access 'https://github.com/jinfuku/fuku-app.git/': Recv failure: Connection was reset
```

这是网络连接问题，常见原因：
1. 网络不稳定
2. 防火墙或代理拦截
3. Git 缓冲区太小
4. GitHub 连接超时

## 💡 解决方案

### 方案一：增加 Git 缓冲区大小（推荐先试）

#### 1.1 执行命令

在 PowerShell 中执行：

```powershell
git config --global http.postBuffer 524288000
```

这会将 Git 的 HTTP 缓冲区设置为 500MB。

#### 1.2 重新推送

```powershell
cd c:\Users\jinfu\WorkBuddy\Claw\fuku-app
git push -u origin main
```

---

### 方案二：使用 SSH 方式（更稳定）

#### 2.1 切换远程仓库为 SSH

```powershell
cd c:\Users\jinfu\WorkBuddy\Claw\fuku-app
git remote set-url origin git@github.com:jinfuku/fuku-app.git
```

#### 2.2 配置 SSH 密钥（如果还没有）

**检查是否有 SSH 密钥：**

```powershell
ls ~/.ssh
```

如果看到 `id_rsa.pub` 或 `id_ed25519.pub`，说明已有密钥。

**生成新的 SSH 密钥（如果没有）：**

```powershell
ssh-keygen -t ed25519 -C "jinfuku@github.com"
```

按提示操作：
- 可以直接按回车使用默认路径
- 可以设置密码（也可以留空）

#### 2.3 添加 SSH 密钥到 GitHub

**1. 复制公钥：**

```powershell
cat ~/.ssh/id_ed25519.pub
```

或打开文件：
```
C:\Users\jinfu\.ssh\id_ed25519.pub
```

复制整个内容（从 `ssh-` 开头到结尾）

**2. 添加到 GitHub：**
- 访问：https://github.com/settings/ssh/new
- 在 "Key" 框中粘贴公钥
- 在 "Title" 中输入：`WorkBuddy`
- 点击 "Add SSH key"

#### 2.4 测试 SSH 连接

```powershell
ssh -T git@github.com
```

如果看到：
```
Hi jinfuku! You've successfully authenticated...
```

说明 SSH 配置成功！

#### 2.5 使用 SSH 推送

```powershell
cd c:\Users\jinfu\WorkBuddy\Claw\fuku-app
git push -u origin main
```

**注意：** 使用 SSH 方式时，不需要输入用户名和密码！

---

### 方案三：使用代理（如果需要）

#### 3.1 配置 Git 使用代理

**如果您使用 VPN 或代理：**

```powershell
# HTTP 代理
git config --global http.proxy http://127.0.0.1:7890

# 或 SOCKS5 代理
git config --global http.proxy socks5://127.0.0.1:7890
```

#### 3.2 推送

```powershell
git push -u origin main
```

#### 3.3 取消代理（如果不需要）

```powershell
git config --global --unset http.proxy
```

---

### 方案四：检查网络连接

#### 4.1 测试 GitHub 连接

在 PowerShell 中执行：

```powershell
Test-NetConnection github.com -Port 443
```

如果返回 `TcpTestSucceeded : True`，说明连接正常。

#### 4.2 测试 DNS 解析

```powershell
Resolve-DnsName github.com
```

确认 GitHub 的 IP 地址正确。

#### 4.3 清除 DNS 缓存

```powershell
Clear-DnsClientCache
```

---

### 方案五：重试几次（网络波动）

有时网络波动会导致连接重置，可以多试几次：

```powershell
cd c:\Users\jinfu\WorkBuddy\Claw\fuku-app
git push -u origin main
```

如果失败，等待几秒后重试。

## 🆘 常见问题

### Q1: 增加缓冲区后还是失败？

**解决方案：**
1. 尝试切换到 SSH 方式
2. 检查是否有防火墙拦截
3. 尝试关闭 VPN 或代理
4. 等待网络稳定后再试

### Q2: SSH 推送也失败？

**错误信息：**
```
ssh: connect to host github.com port 22: Connection timed out
```

**解决方案：**
1. 某些网络会阻止 SSH 端口（22）
2. 尝试使用 HTTPS 方式 + 增加缓冲区
3. 检查防火墙设置

### Q3: 如何切换回 HTTPS 方式？

```powershell
git remote set-url origin https://github.com/jinfuku/fuku-app.git
```

### Q4: 推送成功但网站还是 404？

**原因：** GitHub Pages 还没有配置

**解决：**
1. 访问：https://github.com/jinfuku/fuku-app/settings/pages
2. 配置：
   - Source: Deploy from a branch
   - Branch: main
   - Directory: / (root)
3. 点击 Save
4. 等待 1-3 分钟
